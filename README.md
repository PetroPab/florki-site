# florki.ru

Сайт студии флорариумов **«Флорки»** в Ярославле.

Стильные композиции с живыми растениями в стекле —
готовые работы в наличии и под индивидуальный заказ.

---

## 🌿 О проекте

«Флорки» — локальная студия флорариумов в Ярославле.
Сайт-витрина для демонстрации работ, приёма заявок
и формирования доверия через отзывы и полезные статьи.

Заказ обрабатывается без онлайн-оплаты — клиент
выбирает удобный канал связи (Telegram, WhatsApp,
звонок или форма) и общается со студией напрямую.

## 📑 Разделы сайта

- **Главная** — Hero, преимущества, топ-композиции,
  блок «Под заказ», избранные отзывы, свежие статьи
- **Каталог** — товары в наличии (приоритет) и под заказ
- **Карточка товара** — фото, описание, CTA «Заказать»
- **Галерея** — фотографии работ с lightbox
- **Отзывы** — отдельная страница с фильтром по источникам
  (Авито, Яндекс.Карты, 2ГИС и др.)
- **Полезное** — статьи об уходе и композициях (MDX, SEO)
- **Контакты** — адрес, телефон, карта, форма

## 🎨 Художественное направление

**Botanical Glassmorphism** — стеклянные слои интерфейса
как метафора стеклянной витрины оранжереи. Тёплая
ботаническая палитра, гуманистический сериф Fraunces,
живые фотографии композиций как главные герои.

## 🛠 Стек

- **Next.js 14** (App Router) + **TypeScript**
- **Tailwind CSS** + **Framer Motion**
- **Lucide React** — иконки
- **React Hook Form** + **Zod** — валидация формы
- **Resend** — отправка писем
- **MDX** — статьи раздела «Полезное»
- **Embla Carousel** — карусель отзывов
- **Radix UI Dialog** — модалка выбора канала связи
- Деплой: **Vercel**

## 🚀 Локальный запуск

```bash
# Установка
npm install

# Разработка
npm run dev
# → http://localhost:3000

# Сборка для прода
npm run build

# Локальный прод-режим
npm run start

# Проверка типов и линтинга
npm run lint
npm run typecheck
```

## 🌐 Деплой

Автоматический деплой на **Vercel** при пуше в `main`.

- Production: `florki.ru`
- Preview: каждый PR получает свой URL вида
  `florki-site-{branch}.vercel.app`

## 🔐 Переменные окружения

Создать `.env.local` на основе `.env.example`:

