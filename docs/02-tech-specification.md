# 02. Техническое задание

Главный документ по функциональности сайта
florki.ru. При расхождениях с другими файлами —
приоритет у этого ТЗ.

---

## 1. Общие требования

### 1.1 Платформа

- **Next.js 14** (App Router)
- **TypeScript** strict mode
- **Tailwind CSS** для стилей
- **Vercel** для хостинга
- **Домен:** florki.ru

### 1.2 Языки и локализация

- Только **русский** язык
- `lang="ru"` в `<html>`
- Даты — формат `DD.MM.YYYY` или «5 марта 2024»

### 1.3 Темы

- Только **светлая** тема
- Без переключателя
- Без `prefers-color-scheme: dark` override

### 1.4 Брейкпоинты

- Mobile: 320–767px (базовый)
- Tablet: 768–1023px
- Desktop: 1024–1279px
- Wide: 1280px+

Подход: **mobile-first**.

### 1.5 Браузеры

- Chrome / Edge — последние 2 версии
- Safari — последние 2 версии (iOS включительно)
- Firefox — последняя версия
- Samsung Internet — последняя

IE11 — **не поддерживаем**.

## 3. Страницы — детально

### 3.1 Главная `/`

1. **Hero**

2. **USP-блок (4 преимущества)**

   - CTA «Обсудить идею»

   - 3–5 избранных отзывов с разных площадок

6. **Полезное (3 свежие статьи)**
   - Превью статей с обложкой

7. **FAQ (опционально, аккордеон)**
   - 5–7 частых вопросов

   - Большой заголовок «Сделаем флорариум под вас»
   - CTA «Написать»

### 3.2 Каталог `/catalog`

**Структура:**

- H1: «Каталог флорариумов»
- Подзаголовок: коротко про каталог
- Фильтр-табы: **Все / В наличии / Под заказ**

- Сетка карточек: 1 (mobile) / 2 (tablet) / 3 (desktop)

**Карточка товара:**

- Фото (4:5 или 1:1, обязательно)
- Бейдж: «В наличии» (зелёный) или «Под заказ» (янтарь)
- Название
- Габариты (Ø или ВхШ)
- Цена (или «от X ₽» для «под заказ»)
- CTA «Подробнее»

**Без пагинации** на старте — все товары на одной
странице (товаров будет 15–40, это нормально).

**Пустой каталог:** заглушка с CTA «Под заказ».

### 3.3 Карточка товара `/catalog/[slug]`

Под основным блоком:

- Описание (2–4 абзаца, опционально)
- Блок «Похожие работы» (3 карточки)

**SEO:**

- Title: «{название} — флорариум в Ярославле»

- JSON-LD `Product`

**Структура:**

- H1: «Наши работы»

### 3.5 Отзывы `/reviews`

**Структура:**

- H1: «Отзывы клиентов»
- Подзаголовок про честность отзывов
- Фильтр по площадкам: **Все / Авито / Яндекс / 2ГИС /

- Сетка карточек отзывов: 1 / 2 / 3 колонки

**Карточка отзыва:**

- Лого площадки + название
- Звёзды (если есть)
- Имя автора (или ник)
- Дата

- Фото от клиента (если есть, опционально)

**Без сбора отзывов через сайт.** Только импорт
из админских правок (см. `content/reviews/`).

**Структура:**

- H1: «Полезное о флорариумах»

### 3.8 Контакты `/contacts`

**Структура (2 колонки на десктопе):**

Левая колонка:

- H1: «Контакты»
- Адрес студии в Ярославле
- Телефон (clickable `tel:`)
- Telegram, WhatsApp (clickable ссылки)
- Email
- Часы работы
- Карта (Яндекс.Карты, embed)

Правая колонка:

- Форма «Написать студии»:
  - Имя (required)
  - Контакт: телефон или email (required, либо–либо)
  - Сообщение (textarea, required, min 10 символов)
  - Чекбокс согласия с политикой (required)
  - Honeypot-поле (антиспам)
  - reCAPTCHA v3 (опционально)
- После отправки: тост успеха + сброс формы

**Отправка:**

- API route: `app/api/contact/route.ts`
- Сервис: **Resend** (письмо на `CONTACT_EMAIL`)
- Тема письма: «Заявка с florki.ru — {имя}»
- Логирование: только сам факт отправки (без PII)

- Возврат и обмен (по закону «О защите
  прав потребителей»)

## 4. Функциональные блоки

### 4.1 Модалка заказа (OrderModal)

**Триггеры открытия:**

- CTA «Заказать» на карточке товара
- CTA «Заказать» на главной (USP-блок, под заказ)
- Floating CTA «Написать» (десктоп)

**Содержимое:**

- Заголовок: «Как вам удобнее связаться?»
- 4 варианта (карточками):
  1. **Telegram** — «Ответим за 15 минут»
     (основной, выделен)
  2. **WhatsApp** — «Ответим за 15 минут»
  3. **Звонок** — «10:00–20:00 ежедневно»
  4. **Форма** — «Напишем на email»

**Логика ссылок:**

```ts
// Telegram
`https://t.me/${USERNAME}?text=${encodeURIComponent(msg)}`
// WhatsApp
`https://wa.me/${PHONE}?text=${encodeURIComponent(msg)}`
// Звонок
`tel:${PHONE}`;

// Форма
router.push('/contacts#form');
```

**Предзаполнение сообщения:**

- С карточки товара:
  `«Здравствуйте! Интересует флорариум "{название}" 
(florki.ru/catalog/{slug})»`
- С главной/общего CTA:
  `«Здравствуйте! Хочу обсудить флорариум на заказ»`

**Реализация:**

- Radix UI Dialog
- Управление через `useOrderModal()` zustand-стор
  (или React Context, если простой)

### 4.2 Lightbox

Используется в:

- Карточка товара (галерея)
- Галерея `/gallery`
- MDX `<Gallery>` в статьях

**Возможности:**

- Открытие по клику на превью
- Закрытие: Esc, клик вне, крестик
- Навигация: ← →, свайпы на тач
- Zoom: pinch на тач, scroll на десктопе (опционально)
- Анимация открытия (Framer Motion `layoutId`)

**Реализация:** свой компонент или
**yet-another-react-lightbox**.

### 4.3 Карусель отзывов на главной

- **Embla Carousel** (React)
- 1 / 2 / 3 видимых на mobile / tablet / desktop
- Точки-индикаторы снизу
- Стрелки на десктопе
- Свайпы на тач
- Autoplay: **нет** (пользователь сам управляет)

### 4.4 Cookie-баннер

**Минимальная версия:**

- Появляется через 1 сек после загрузки
- Текст: «Используем cookie для аналитики.
  Подробнее в [Политике](/privacy)»
- 1 кнопка: «Понятно»
- Закрытие → флаг в `localStorage`, больше не показывать

**Не сложнее.** Без галочек, без выбора категорий,
без GDPR-зоопарка.

## 5. Контент

### 5.1 Источники

| Тип    | Где лежит                | Формат              |
| ------ | ------------------------ | ------------------- |
| Товары | `content/catalog/*.ts`   | TypeScript          |
| Отзывы | `content/reviews/*.ts`   | TypeScript          |

| Фото   | `public/images/...`      | WebP + JPG fallback |

### 5.2 Структура товара (тип)

```ts
type Product = {
  slug: string; // 'mshistyi-les'
  title: string; // 'Мшистый лес'
  status: 'in-stock' | 'on-order';
  price: number; // в рублях, без копеек
  priceFrom?: boolean; // true → «от 3 500 ₽»
  size: {
    diameter?: number; // см
    width?: number;
    height?: number;
  };
  plants: string[]; // ['мох', 'хедера', 'фиттония']
  care: string; // 1–2 предложения
  description?: string; // 2–4 абзаца (Markdown)
  images: {
    src: string;
    alt: string;
  }[];
  mainImage: number; // индекс главного фото
  createdAt: string; // ISO дата для сортировки
};
```

### 5.3 Структура отзыва (тип)

```ts
type Review = {
  id: string;
  source: 'avito' | 'yandex' | '2gis' | 'telegram' | 'instagram' | 'other';

  author: string;
  rating?: 1 | 2 | 3 | 4 | 5;
  date: string; // ISO
  text: string;
  photos?: { src: string; alt: string }[];
  featured?: boolean; // показывать на главной
};
```

## 6. SEO

### 6.1 Базовые требования

- Уникальные `<title>` и `<meta description>`
  на каждой странице
- Open Graph + Twitter Card теги
- `sitemap.xml` (генерация автоматическая)
- `robots.txt` (разрешить всё, кроме /api/)
- `lang="ru"` в HTML
- Канонические URL (`<link rel="canonical">`)

### 6.2 Структурированные данные (JSON-LD)

- Главная: `LocalBusiness` с адресом и часами
- Карточка товара: `Product` с offers
- Статья: `Article` с autor, date, image
- Хлебные крошки: `BreadcrumbList`

### 6.3 Ключевые запросы

Приоритет:

1. флорариум Ярославль
2. купить флорариум Ярославль
3. флорариум на заказ Ярославль
4. флорариум в стекле подарок

Семантика разнообразных запросов раскрывается
в статьях `/poleznoe`.

### 6.4 Производительность для SEO

- **Все страницы — SSG** (статическая генерация)
- ISR не используем (контент меняется редко)
- LCP < 2.5s, FID < 100ms, CLS < 0.1

## 9. Доступность (WCAG 2.1 AA)

### 9.1 Обязательно

- Контраст текста ≥ 4.5:1 (обычный)
- Контраст текста ≥ 3:1 (крупный, 18pt+)
- Все интерактивы доступны с клавиатуры
- Visible focus rings (не убирать `outline` без замены)
- Touch target ≥ 44×44px
- Alt-тексты на всех `<img>`
- Aria-labels на иконках-кнопках
- `prefers-reduced-motion: reduce` → анимации off/min
- Семантические теги (`<nav>`, `<main>`, `<article>` …)

### 9.2 Skip-to-content

В хедере — скрытая ссылка «Перейти к содержимому»,
видимая при фокусе.

## 10. Безопасность

### 10.1 Headers (Vercel-конфиг)

```
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=()
```

### 10.2 Форма /contacts

- **Rate limit** на API route: 5 запросов / 10 минут
  с одного IP (Vercel KV или `upstash/ratelimit`)
- **Honeypot-поле** `website` (скрытое, должно быть пусто)
- **Валидация на сервере** через Zod
- **Очистка XSS** на входе (хотя текст не рендерится
  как HTML)

### 10.3 .env

Никогда не коммитить `.env.local`. Только `.env.example`
с пустыми значениями.
