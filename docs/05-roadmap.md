# 05. Дорожная карта

7 этапов от пустой папки до запущенного сайта.
Реалистичный срок: **6–8 недель**.

---

## Шкала

| Этап          | Срок      | Результат                 |
| ------------- | --------- | ------------------------- |
| 0. Подготовка | 2–3 дня   | Контент, фото, доступы    |
| 1. Фундамент  | 2–3 дня   | Репо, токены, Vercel      |
| 2. UI-kit     | 3–5 дней  | Базовые компоненты        |
| 3. Страницы   | 7–10 дней | Все страницы с заглушками |
| 4. Контент    | 3–5 дней  | Реальные тексты и фото    |
| 5. Полировка  | 3–5 дней  | Анимации, SEO, a11y, perf |
| 6. Запуск     | 1–2 дня   | Домен, аналитика          |

---

## Этап 0. Подготовка

**Контент:**

- Тексты главной, О студии, Контакты
- 8–12 товаров: название, описание, габариты, цена
- 8–15 отзывов с площадок (со ссылками)
- 3–5 черновиков статей
- Политика, оферта, реквизиты

**Фото:** 15–20 работ + hero + студия +
для статей. JPG ≥ 2000px, на Google Drive.

**Доступы:** домен florki.ru, Vercel, GitHub,
Метрика, Telegram, почта hello@florki.ru.

**Готово, когда:** материалы можно передать
разработчику без вопросов.

---

## Этап 1. Фундамент

- `create-next-app` с TS + App Router + Tailwind
- ESLint, Prettier, Husky
- Структура папок по `02-tech-architecture.md`
- Дизайн-токены из `03-design-concept.md`
  в `globals.css` + `tailwind.config.ts`
- Шрифты Fraunces/Inter через `next/font`
- Библиотеки: lucide, framer-motion, clsx,
  next-mdx-remote, zod
- Vercel preview-деплои на каждый PR

**Готово, когда:** `npm run dev` запускается,
шрифты подключены, ESLint молчит.

---

## Этап 2. UI-kit

**Базовые (`/components/ui/`):**
Button, Badge, Card, Input, Textarea, Checkbox,
Container, Heading, Text, Link.

**Композитные (`/components/blocks/`):**
ProductCard, ReviewCard, ArticleCard, GlassPanel,
Callout.

**Layout (`/components/layout/`):**
Header (sticky + glass на скролле), MobileMenu
(drawer), Footer, Section.

Все компоненты адаптивны, доступны с клавиатуры,
имеют focus states.

**Готово, когда:** любую страницу можно собрать
из готовых компонентов без новой вёрстки.

---

## Этап 3. Страницы

**Главная:** Hero, USP, featured-товары,
блок «Под заказ», отзывы, CTA.

**Каталог `/katalog`:** сетка карточек, фильтр
по статусу.

**Карточка `/katalog/[slug]`:** галерея + lightbox,
габариты, состав, описание, уход, CTA, похожие.

**Полезное `/poleznoe`:** сетка статей + фильтр
по тегам.

**Статья `/poleznoe/[slug]`:** MDX-рендер,
TOC на десктопе, CTA, похожие.

**О студии, Контакты** (адрес, часы, форма, карта).

**Правовые** `/legal/privacy`, `/legal/oferta`.

**Служебные:** 404, 500, loading-скелетоны.

**Готово, когда:** все страницы существуют,
навигация работает, нет 404.

---

## Этап 4. Контент

- Товары → `/data/products.json` или MDX
- Фото → `/public/images/products/` с alt-текстами
- Статьи → `/content/articles/` с frontmatter
- Отзывы → `/data/reviews.json` со ссылками
- Финальные тексты главной, «О нас», контактов
- Политика, оферта
- OG-картинки (дефолтная 1200×630)

**Готово, когда:** нет Lorem ipsum и TODO,
все картинки оптимизированы (WebP, <200KB).

---

## Этап 5. Полировка

**Анимации:** появление при скролле, hover карточек,
модалка заказа, lightbox через `layoutId`,
проверка `prefers-reduced-motion`.

**SEO:** metadata на каждой странице, sitemap.ts,
robots.txt, Open Graph, JSON-LD (Organization,
Product, Article), canonical.

**A11y:** клавиатурная навигация, focus states,
контрасты AA, alt у всех картинок, иерархия
заголовков, ARIA на модалках/меню.

**Performance:** `next/image` везде, lazy-loading,
Lighthouse Performance ≥ 90 mobile, LCP < 2.5s,
CLS < 0.1.

**Формы:** Server Action + Zod, honeypot +
rate limit, отправка через Resend на
hello@florki.ru.

**Тестирование:** Chrome, Firefox, Safari + iOS/Android.

**Готово, когда:** Lighthouse Perf ≥ 90,
A11y ≥ 95, SEO = 100.

---

## Этап 6. Запуск

- Домен florki.ru → Vercel (DNS + SSL)
- Редирект www → без www (или наоборот)
- Яндекс.Метрика + цели (Telegram/WhatsApp/форма)
- Подтверждение в Яндекс.Вебмастере + sitemap
- Финальный прогон: ссылки, телефоны, форма, OG
- Тег `v1.0.0` в Git, бэкап
- Краткая инструкция для основательницы
  (как добавить товар/статью)

**Готово, когда:** сайт открывается на florki.ru,
аналитика работает, клиент знает, как обновлять.

| Когда         | Что показываем            | Кому                         |
| ------------- | ------------------------- | ---------------------------- |
| После Этапа 2 | UI-kit                    | Арт-директор, основательница |
| После Этапа 3 | Все страницы с заглушками | Основательница               |

---

## Главные риски

1. **Контент не готов** → не начинаем Этап 4 без него.
   В крайнем случае стартуем с 4–5 товарами.
2. **Бесконечные правки дизайна** → дизайн-концепция
   (док. 03) — закон. Правки через PR.
3. **Технический долг** → code review обязателен,
   прямые пуши в `main` запрещены.
4. **Падает производительность** → Lighthouse
   проверяем на каждом этапе, не в конце.
5. **«Ещё чуть-чуть» перед запуском** → v1.0 —
   фиксированный scope, всё новое — в backlog v1.1.

## После запуска

**v1.1+ (когда будет смысл):** больше товаров,
видео процесса, корпоративный раздел, тёмная тема,
онлайн-оплата.

**Регулярка:**

- Раз в неделю — новые отзывы
- Раз в две недели — новый товар
- Раз в месяц — новая статья
- Раз в квартал — аудит perf/SEO
- Раз в год — обновление зависимостей

**Метрики успеха через 3 месяца:**

- Индексация в Яндексе
- 50+ уникальных посетителей в день
- Топ-3 по «флорариум Ярославль»
- 10+ заявок в месяц через сайт
