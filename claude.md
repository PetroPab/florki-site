Этот файл — точка входа для AI-ассистента (Claude Code

## 🌿 Контекст проекта

**Что:** сайт студии флорариумов «Флорки» в Ярославле.  
**Зачем:** продавать готовые композиции, принимать
заказы на индивидуальные работы, формировать доверие.  
**Кому:** жители Ярославля 25–45 лет, ищущие
оригинальный подарок или декор.

**Художественное направление:** Botanical Glassmorphism
— стеклянные слои + тёплая ботаническая палитра +
крупные фотографии композиций.

## 🛠 Стек

- **Next.js 15** (App Router, Turbopack) + **TypeScript** (strict)
- **Tailwind CSS v3** + **Framer Motion** (точечно, только где нужна JS-анимация)
- **Lucide React** — иконки
- **React Hook Form** + **Zod** — формы и валидация
- **Resend** — отправка писем со страницы /kontakty
- **MDX** + **next-mdx-remote** — статьи /poleznoe
- **Radix UI Dialog** — модалка выбора канала связи
- **nextjs-toploader** — прогресс-бар при навигации
- **Vercel** — хостинг и деплой
- **Домен:** florki.ru

**Шрифты:**

- **Fraunces** — display (заголовки, variable, italic для `.display-em`)
- **Golos Text** — body (основной текст, кириллица)

**Ключевые CSS-паттерны:**

- `.display-em` — italic Fraunces для акцентных слов в заголовках
- `.heading-rule` / `.heading-rule-center` — размытый овальный орнамент под заголовком
- `.glass-section` / `.glass-section-dense` — glassmorphism фоны секций
- `.btn-shimmer` — hover-блик на кнопках
- `.btn-pulse` — пульсирующее кольцо на главных CTA
- `[data-reveal]` / `.revealed` — scroll-reveal через IntersectionObserver + CSS

   там зафиксированы все решения.
2. Применять Quick Reference §1–§9 из `ui-ux-pro-max`.

   никаких generic-шрифтов, никаких generic-палитр.

### Команда генерации/обновления design-system

```bash
python3 skills/ui-ux-pro-max/scripts/search.py \
  "florarium plants studio glassmorphism botanical warm" \
  --design-system --persist \
  -p "Florki" \
  -f markdown
```

### Специфика glassmorphism в этом проекте

Стекло — **смысловая метафора**, не декор. Правила:

- **Контраст текста на стекле** проверяется отдельно —
  фон под стеклом может быть тёмным (фото). Минимум
  4.5:1 в худшем сценарии.
- **Fallback** для браузеров без `backdrop-filter`
  обязателен: `@supports not` блок с плотным фоном.
- **Бюджет:** не более 5–7 стеклянных элементов
  одновременно на экране (производительность GPU).
- **На мобильных** — упрощать blur (12px вместо 20px).
- **Не везде:** длинные тексты статей, футер, инпуты
  форм — без стекла.

## 🎯 Архитектурные принципы

### Контент в файлах

Весь контент (товары, отзывы, статьи) — TypeScript-
или MDX-файлы. **Никаких БД и CMS.** Это позволяет:

- Версионировать контент в git
- Быстро деплоить через Vercel
- Не платить за хостинг БД

### Структура контента

```
content/
├── catalog/           ← товары (TS)
│   ├── mshistyi-les.ts
│   └── ...
├── reviews/           ← отзывы (TS)
│   ├── 001-anna-k.ts
│   └── ...
└── articles/          ← статьи (MDX)
    ├── kak-uhazhivat-za-florariumom.mdx
    └── ...
```

### Типы данных

Все типы — в `types/`:

- `types/product.ts` — товар каталога
- `types/review.ts` — отзыв
- `types/article.ts` — статья

### Утилиты

- `lib/catalog.ts` — загрузка и сортировка товаров
- `lib/reviews.ts` — загрузка и фильтрация отзывов
- `lib/articles.ts` — загрузка статей и парсинг MDX
- `lib/order-links.ts` — генерация ссылок
  TG/WA/tel с предзаполненным текстом

## 💬 Механика заказа

Сайт **не имеет корзины**. Вместо этого:

1. Клиент видит CTA «Заказать»
2. Открывается модалка с выбором канала:
   **Telegram (основной) / WhatsApp / Звонок / Форма**
3. При выборе мессенджера — переход в чат
   с **предзаполненным сообщением** про конкретный товар

Реализация — в `lib/order-links.ts` + компонент
`OrderModal`. Конфигурация — через `NEXT_PUBLIC_*`
переменные окружения.

## 🗣 Tone of Voice

В текстах сайта и коммуникации:

- **Тепло, дружелюбно** — не корпоративно
- **Просто** — без снобизма и эзотерики
- **На «вы»** — но без официоза
- **Конкретно** — «живут 2–3 года», а не
  «дарят радость на долгие годы»

**Чего избегаем:**

- «Эзотерический» язык: «энергия», «гармония»,
  «дзен», «медитация»
- Канцелярит: «осуществляем реализацию композиций»
- Заискивание: «наши дорогие клиенты»

## 🤖 Правила работы AI-ассистента

1. **Не выдумывать контент.** Если нужен текст товара,
   отзыва или статьи — спросить или взять из
   `content/` / `docs/04-content-plan.md`.
2. **Не нарушать запреты** из секции выше. Если
   кажется, что нужен Inter/тёмная тема/градиент —
   значит, есть лучшее решение.
3. **Сверяться с MASTER.md** при работе над UI.
   Если компонента там нет — добавить туда после
   создания.
4. **Не использовать `any`** в TypeScript.
   `strict: true` включён намеренно.
5. **Не коммитить `.env.local`** — только `.env.example`.
6. **Спрашивать, если непонятно.** Лучше уточнить,
   чем сделать не то.
